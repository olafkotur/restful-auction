API Overview

  • List all auctions
  • Add a new auction
  • Find an auction
  • Update an auction
  • Delete an auction
  • Place a bid
  • List auction bids
  • Create user
  • List all users
  • Login user

GET /api/auctions

Description: List all auctions

Media Type: application/json

Headers: None

Parameters: No Parameters

Responses:
  • 200: Successful operation

POST /api/auction

Description: Add a new auction

Media Type: application/json

Headers: Authorization - JWT Bearer token

Parameters:

  • name string
  • firstBid float
  • sellerId integer
  • reservePrice float optional
Responses:
  • 200: Successful operation
  • 405: Invalid input

GET /api/auction{auctionId}

Description: Find auction by ID

Media Type: application/json

Headers: None

Parameters:

  • auctionId integer
Responses:
  • 200: Successful operation
  • 400: Invalid ID supplied
  • 404: Auction not found

POST /api/auction{auctionId}

Description: Updates an auction in the store with form data

Media Type: application/json

Headers: Authorization - JWT Bearer token

Parameters:

  • name string
  • firstBid float
  • sellerId integer
Responses:
  • 200: Successful operation
  • 400: Invalid ID supplied
  • 404: Auction not found
  • 405: Invalid input

DELETE /api/auction{auctionId}

Description: Deletes an auction

Media Type: application/json

Headers: Authorization - JWT Bearer token

Parameters: No Parameters

Responses:
  • 200: Successful operation
  • 400: Invalid ID supplied
  • 404: Auction not found
  • 405: Invalid input

POST /api/auction{auctionId}/bid

Description: Place a bid for an auction

Media Type: application/json

Headers: Authorization - JWT Bearer token

Parameters:

  • bidAmount float
  • bidderId integer
Responses:
  • 200: Successful operation
  • 400: Invalid ID supplied
  • 404: Auction not found
  • 405: Invalid input

GET /api/auction{auctionId}/bids

Description: Lists bids for a particular auction

Media Type: application/json

Headers: Authorization - JWT Bearer token

Parameters: No Parameters

Responses:
  • 200: Successful operation
  • 405: Invalid input

POST /api/user

Description: Create user

Media Type: application/json

Headers: None

Parameters:

  • username string
  • password string
Responses:
  • 200: Successful operation
  • 400: Invalid username/password supplied

POST /api/users

Description: Lists all usernames and ids

Media Type: application/json

Headers: None

Parameters: No Parameters

Responses:
  • 200: Successful operation

POST /api/user/login

Description: Logs user into the system

Media Type: application/json

Headers: None

Parameters:

  • username string
  • password string
Responses:
  • 200: Successful operation
  • 400: Invalid username/password supplied
